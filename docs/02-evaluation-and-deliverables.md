# 02 — Evaluation Criteria & Deliverables

## CrimeGPT — How You're Judged

From PS-69EEFDFB90B99 evaluation criteria:

| Criterion | What they actually want to see |
|-----------|-------------------------------|
| Accuracy of document generation | Consistent, correct docs from shared input, minimal duplication/errors |
| Legal section mapping intelligence | Correct BNS/BNSS/BSA sections + relevant case law from incident narrative |
| UX for non-technical officers | Clear workflow, easy navigation, multilingual |
| Case diary integration | Structured chronological diary from first report to arrest |
| Language support | Quality of Gujarati/Hindi/English I/O for real deployment |
| Deliverable completeness | Working solution + docs + code structure + dataset + demo |
| Innovation | Offline mode, CCTNS/BharatPol integration, face recognition, evidence tracking |

### CrimeGPT Required Deliverables
1. Working prototype with **at least 4 auto-generated documents**.
2. Demo showing: case creation FIR→arrest, real-time generation of ≥2 documents,
   legal section + judgment suggestions.
3. Documentation (README, user guide, code).
4. Dataset used (anonymized legal texts, FIR samples).

## Universal Mandatory Deliverables (both PSs)

- **Abstract** (solution + methodology + tools) — gating, disqualification if missing.
- **Presentation** — demoable, in-depth — gating, disqualification if missing.

---

## Our Deliverable Build Order (recommended)

1. **Abstract** for each PS (1 page) — write first, lock the scope.
2. **Architecture diagram** for each PS — anchors the pitch.
3. **Core demo path** — the single end-to-end flow that must work flawlessly.
4. **Compliance layer** — the differentiator (hashing, certificates, consent dashboard).
5. **Presentation deck** — built around the working demo, not aspirational features.
6. **Documentation / README** — last, describing what was actually built.
